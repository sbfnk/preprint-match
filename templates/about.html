About Preprint Match

Every week, hundreds of new preprints appear on medRxiv. Each one will eventually find its way to a journal — or not. We predict where.

What this is

Preprint Match uses machine learning to predict which journal a medRxiv preprint is most likely to be published in. We train on {{ "{:,}".format(meta.n_papers_training) }} preprints that have already been published, learning the patterns that connect papers to journals: topic, methodology, scope, writing style.

For each of the {{ "{:,}".format(meta.n_papers) }} current preprints on medRxiv, we compute calibrated probability estimates across {{ meta.n_journals }} journals with sufficient training data. You can explore this from either direction: pick a paper and see where it might land, or pick a journal and see what's coming. Preprints destined for niche journals outside this set will still get predictions, but among the journals we do cover.

Why

Journals are filters. They take a stream of research and select what fits their scope, standards, and audience. That has its utility — but it can also be slow, opaque, and concentrated in the hands of a few publishers.

Preprints changed the first part: anyone can share their work immediately. But discovery is still hard. If you work in infectious disease epidemiology, how do you find the preprints that matter to you among thousands posted each month?

We think the answer looks less like a journal and more like a personalised feed — one that understands your interests and surfaces relevant work as it appears, not months later when it's finally published.

Where this is going

  • Custom journals — Define your own scope. Combine topics, methods, and populations to create a feed that no existing journal covers. Get notified when a matching preprint appears.
  • Beyond medRxiv — bioRxiv, arXiv, and other preprint servers have the same discovery problem. The approach generalises.
  • Better recommendations — Journal prediction is a stepping stone. The real goal is connecting researchers with the work they need to see, regardless of where (or whether) it gets published.
  • Linking with peer review — Preprints don't have to wait for journals to be reviewed. Platforms like Peer Community In, PREreview, Rapid Reviews: Infectious Diseases, and eLife are already reviewing preprints openly. Surfacing those evaluations alongside predictions is a natural next step.

How it works

We train on {{ "{:,}".format(meta.n_papers_training) }} medRxiv preprints that were subsequently published in peer-reviewed journals. Each paper is encoded into a 768-dimensional vector using SPECTER2, a scientific document embedding model trained on citation graphs. We fine-tune the model's adapter layer using contrastive learning: papers published in the same journal are pulled together in embedding space, while papers from different journals are pushed apart. To make the task harder (and the embeddings better), training batches group papers from the same medRxiv category, forcing the model to learn fine-grained distinctions rather than broad topical ones.

Predictions combine two approaches: a nearest-neighbour lookup (which papers in the training set are most similar?) and a logistic regression classifier (which journal do the embedding features point to?). The two are blended with a 90/10 classifier/kNN weight, tuned on a held-out validation set. Probabilities are calibrated using isotonic regression so that when the model says "15% chance of PLOS ONE", roughly 15% of such predictions are correct.

On our test set (6,950 papers, 316 journals), the model gets the exact journal right 20% of the time, and the correct journal appears in the top 10 predictions 61% of the time. For the 20 most common journals, the top-10 list is correct 82% of the time.

The full methodology — dataset construction, embedding comparison, fine-tuning details, per-tier evaluation, calibration analysis — is described in RESULTS.md. All code is open source.
